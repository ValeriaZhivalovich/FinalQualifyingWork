**ДИПЛОМНЫЙ ПРОЕКТ**

**Анализ новостных лент на основе ИИ-агентов и методов НЛП**

Стек: Python 3.11 · Flet · SQLite · Ollama · Telethon · vk_api · feedparser


# **1\. Цель и границы проекта**
Цель проекта - разработать десктопное приложение, которое автоматически собирает публикации из социальных сетей и новостных источников, обрабатывает их с помощью ИИ-агентов и методов обработки естественного языка (NLP), генерирует краткое резюме каждой новости и присваивает ей категорию, после чего сохраняет результат в базу данных для последующего просмотра через графический интерфейс(приложение на десктоп и мобильной вариации).

## **1.1 Что входит в проект (In Scope)**
- Парсинг публикаций из Telegram-каналов, VK-сообществ и RSS-лент
- Модульная архитектура коннекторов: каждый источник - отдельный независимый модуль
- NLP-обработка: токенизация, удаление стоп-слов, нормализация текста
- ИИ-агент на базе локальной LLM (Ollama) для генерации резюме и категоризации
- Хранение данных в SQLite через SQLAlchemy ORM
- Графический интерфейс на Flet: просмотр, фильтрация, управление источниками
- Система категорий: политика, технологии, экономика, спорт, культура, прочее
- Дедупликация новостей по хеш-сумме текста

# **2\. Архитектура системы**
Система построена по принципу модульного конвейера (pipeline). Каждый слой не зависит от реализации соседнего - взаимодействие происходит через общие интерфейсы (абстрактные базовые классы Python). Это обеспечивает лёгкое добавление новых источников, замену LLM-бэкенда и независимое тестирование компонентов.

## **2.1 Слои системы**

| **Слой**         | **Компонент**           | **Технология**               | **Ответственность**                       |
| ---------------- | ----------------------- | ---------------------------- | ----------------------------------------- |
| 1\. Сбор данных  | Коннекторы (Collectors) | Telethon, vk_api, feedparser | Получение сырых публикаций из источников  |
| 2\. Нормализация | Text Preprocessor       | NLTK, re, langdetect         | Очистка, токенизация, нормализация текста |
| 3\. ИИ-обработка | AI Agent                | Ollama (mistral / llama3.2)  | Генерация резюме, определение категории   |
| 4\. Хранение     | Database Layer          | SQLite + SQLAlchemy          | Персистентность, дедупликация             |
| 5\. Интерфейс    | Flet UI                 | Flet (Flutter/Python)        | Просмотр, фильтрация, управление          |
| 6\. Оркестрация  | Scheduler / Pipeline    | APScheduler                  | Периодический запуск парсинга             |

## **2.2 Схема потока данных**

**Источник → Коллектор → Препроцессор → ИИ-агент → БД → Flet UI**

Каждая стрелка - вызов метода интерфейса. Коллектор возвращает RawArticle, препроцессор - CleanArticle, ИИ-агент - ProcessedArticle. В БД попадает только ProcessedArticle.

## **2.3 Принцип модульности коллекторов**

Все коллекторы наследуют от единого абстрактного класса BaseCollector:

class BaseCollector(ABC):  
source_name: str # "telegram" | "vk" | "rss" | "twitter"  
def fetch(self) -> list\[RawArticle\]: ...  
def validate_config(self) -> bool: ...

Добавление нового источника (Twitter, YouTube и др.) сводится к созданию нового класса-наследника и его регистрации в CollectorRegistry. Никакой другой код менять не нужно.

# **5\. Описание ключевых компонентов**

## **5.1 Коллекторы**

### **Telegram (Telethon)**

Использует Telethon - официальную Python-библиотеку для Telegram MTProto API. Требует регистрации приложения на my.telegram.org (бесплатно). Позволяет читать публичные и добавленные каналы без ограничений.

- Вход: список username или ID каналов из таблицы Source
- Выход: RawArticle для каждого поста
- Ограничение: ~100 сообщений за запрос, FloodWait обрабатывается автоматически

### **VK (vk_api)**

Использует официальное VK API через библиотеку vk_api. Требует сервисный ключ доступа (бесплатно, без ограничений для чтения публичных сообществ).

- Метод: wall.get для получения постов сообщества
- Ограничение: 3 запроса в секунду - учитывается через rate limiting

### **RSS (feedparser)**

Парсинг стандартных RSS/Atom лент. Не требует аутентификации. Покрывает большинство новостных агентств (РИА Новости, BBC, Lenta.ru и т.д.). Рекомендуется как основной источник для демонстрации работы системы.

- Библиотека: feedparser - стандарт де-факто, поддерживает RSS 0.9, 1.0, 2.0 и Atom
- Ограничений нет - полностью легальный способ получения новостей

## **5.2 NLP Preprocessor**

Последовательный конвейер обработки текста:

- Определение языка (langdetect) - ru/en/другой
- Удаление HTML-тегов, спецсимволов, эмодзи
- Нормализация пробелов, приведение к нижнему регистру
- Токенизация (NLTK punkt_tab)
- Удаление стоп-слов (NLTK stopwords для ru и en)
- Лемматизация (pymystem3 для русского, NLTK WordNet для английского)

Результат: очищенный текст, пригодный для подачи в LLM с минимальным количеством токенов.

## **5.3 AI Agent (Ollama)**

Агент получает очищенный текст и выполняет два последовательных запроса к LLM:

**Запрос 1 - резюме:**  
"Сократи следующую новость до 2-3 предложений  
на русском языке. Только суть. Текст: {text}"  
<br/>**Запрос 2 - категория:**  
"Определи одну категорию для новости из списка:  
политика, технологии, экономика, спорт, культура, прочее.  
Ответь одним словом. Текст: {text}"

Рекомендуемые модели через Ollama:

- mistral:7b - хорошее качество, ~4GB RAM, 5-15 сек/новость на CPU
- llama3.2:3b - быстрее, ~2GB RAM, 2-7 сек/новость на CPU, качество чуть ниже
- qwen2.5:7b - хорошо работает с русским текстом

**⚠** _Ollama устанавливается отдельно (ollama.ai). Приложение проверяет доступность Ollama при запуске и показывает инструкцию, если сервис не запущен._

# **8\. Риски и способы их снижения**

| **Риск**                                  | **Вероятность** | **Способ снижения**                                                              |
| ----------------------------------------- | --------------- | -------------------------------------------------------------------------------- |
| Ollama медленно работает на CPU без GPU   | Высокая         | Использовать llama3.2:3b (меньше), кешировать результаты, показать на скриншотах |
| Telegram FloodWait / блокировка аккаунта  | Средняя         | Парсить не чаще 1 раза в 5 минут, использовать реальный аккаунт с историей       |
| VK API изменит структуру ответа           | Низкая          | Хранить raw_data, тесты с фиксированными fixture-ами                             |
| LLM галлюцинирует категорию               | Средняя         | Валидировать вывод по allowlist категорий, fallback → 'прочее'                   |
| Flet имеет мало документации на русском   | Высокая         | Использовать официальную документацию + Gallery примеры                          |
| Переоценка объёма - не успеть сделать всё | Средняя         | MVP = RSS + ИИ + UI. Остальное - расширения                                      |
